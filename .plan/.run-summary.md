# Active Run Summary

Keep this file short. It should only hold durable conclusions worth loading by
default in future runs for the current run family.

## Decisions

- The Ralph Loop framework is now `.plan`-based and no longer stores its live
  workflow state under SpecKit-branded paths.
- Matt Pocock skills remain external prerequisites, not vendored framework
  assets.
- The installer is interactive-first in v1 and owns both framework-managed
  files and framework-owned skills.
- The default install bundle is `Ralph Loop Core`:
  `luna-grill-with-docs`, `luna-to-prd`, `luna-to-issues`, `commit`, and
  `code-review`.
- Namespaced install paths are part of the contract:
  `.agents/skills/luna-ai/*` for Codex-local and
  `~/.hermes/skills/luna-ai/*` for Hermes-global.
- Planning-phase transitions now use durable repo-local pointers and handoff
  files instead of relying on "latest relevant" directory scans.

## Known Blockers

- Public GitHub bootstrap validation is still pending because the documented `main` bootstrap URL was wrong for the actual remote branch (`master`), and the repository is still not publicly fetchable over raw/codeload endpoints (both public URLs return 404).

## Completed Areas

- Framework starter and helper script surface exist under `frameworks/ralph-loop/`.
- Installer bootstrap and Node package are implemented and locally verified.
- `commit` and `.plan`-based `luna-to-issues` are packaged as installable
  framework-owned skills.
- `luna-to-prd` is the framework-owned PRD wrapper for the `.plan` thin-index
  layout.
- `luna-grill-with-docs` is the framework-owned wrapper that persists live
  grilling sessions into `.plan/grilling/`.
- Canonical phase selectors now exist for the current grilling session, current
  PRD, current grill handoff, and current PRD handoff.
- Consumer rollout is verified against the live `neighborhood-showcase` repo:
  the framework installer ran, repo-local skills landed under
  `.agents/skills/luna-ai/`, legacy workflow state was migrated into `.plan/`,
  and a verification-only Ralph Loop iteration successfully read the migrated
  workspace end-to-end.
- Helper-script automation is now hardened: `sync-state.sh` mutates task/epic/index aggregates in place, `retrieve-history.sh` returns ranked compact bundles across live and archived history, and `archive-run.sh` snapshots the active run family plus canonical pointer targets.
- Runner state-machine enforcement is now live: `runtime-state.sh` selects the active sub-task, persists model-tier/attempt/retrieval/escalation state in `.run-state.json`, and the Hermes/Codex/AGY wrappers feed bounded retrieval bundles plus persisted success/retry/blocked outcomes back through that helper.
