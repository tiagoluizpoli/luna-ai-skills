Keep this file short. It should only hold durable conclusions worth loading by

## Decisions

- The Ralph Loop framework is now `.plan`-based and no longer stores its live
  workflow state under SpecKit-branded paths.
- Matt Pocock skills remain external prerequisites, not vendored framework
  assets.
- The installer is interactive-first in v1 and owns both framework-managed
  files and framework-owned skills.
- The install flow is agents-first (Hermes, Codex, AGY), then a single
  run-level availability mode (local or global). Manual skill-subset selection
  is removed; all framework skills are mandatory in v1.
- Shared framework assets (helper scripts, templates) and agent-specific assets
  (runner scripts) are distinct asset classes. Agent-specific assets are only
  installed for selected agents; unselected agent runner scripts are removed.
- The mandatory skill bundle is `Ralph Loop Core`:
  `luna-grill-with-docs`, `luna-to-prd`, `luna-to-issues`, `commit`, and
  `code-review`.
- Namespaced install paths are part of the contract:
  `.agents/skills/luna-ai/*` for local and
  `~/.hermes/skills/luna-ai/*` for global.
- Installation metadata is recorded in `.plan/.framework-install.json` as a
  generated artifact (not listed in `framework-files.json` managed assets),
  tracking concrete installed assets, selected agents, timestamps, and
  availability details.
- On update, the installer presents previously recorded settings for explicit
  confirmation or override before applying them. Install always asks for agents
  explicitly, even when prior state exists.
- Planning-phase transitions now use durable repo-local pointers and handoff
  files instead of relying on "latest relevant" directory scans.
